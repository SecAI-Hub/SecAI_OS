Your data stays on this device

SecAI OS runs all AI inference locally — on your hardware, using your GPU. No prompts, responses, or model data are sent to any cloud service. The network firewall blocks all outbound connections by default.

What's running right now

Your appliance is running in the offline_private profile.

  • Network: blocked — no data leaves this device
  • Vault: encrypted with AES-256 (LUKS2 + Argon2id)
  • Models: verified through a 7-stage quarantine pipeline before use
  • Services: sandboxed with seccomp-BPF, Landlock, and systemd hardening
  • Audit: tamper-evident hash-chained logs record all security events

What happens if something goes wrong

  • Tampered model detected: automatically quarantined and removed
  • Integrity check fails: system degrades to safe mode, alerts you
  • Suspicious activity: agent frozen, airlock disabled, vault re-locked automatically
  • Bad update: Greenboot rolls back to the last known-good state
  • Worst case: 3-level emergency panic — lock, wipe keys, or full wipe

No telemetry

SecAI OS does not collect any telemetry. No usage analytics, no crash reports, no phone-home, no heartbeat. The only network activity is what you explicitly enable by switching to the "research" or "full lab" profile — and even then, all traffic is Tor-routed with PII stripping.

How to verify

You don't have to take our word for it:

  • The Security dashboard shows real-time verification status
  • Audit logs are hash-chained — any tampering breaks the chain visibly
  • The OS image is cosign-signed with SLSA3 provenance attestation
  • Every model passes 7 stages of automated verification before use
  • Export a forensic bundle for independent verification

For the full technical security model, see the Security dashboard.
